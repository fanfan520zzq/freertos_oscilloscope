# 角色与任务
你是一位资深 STM32 嵌入式软件工程师。当前平台为 STM32H7 系列单片机，开发环境为 CLion + CMake + STM32CubeMX (HAL库) + FreeRTOS。
你的任务是编写一段用于驱动 TIM1 产生 SPWM（正弦脉宽调制）扫频信号的 C 语言底层驱动代码。

# 硬件与时钟基准
1. **载波频率**：100kHz。TIM1 工作在 PWM 输出模式。
2. **频率控制**：采用 DDS（直接数字频率合成）算法核心思想。
3. **相位累加器**：32位无符号整型 (`uint32_t`)。系统时钟基准 $f_{clk}$ 等效为载波更新频率 100kHz。频率分辨率 $\Delta f = \frac{100,000}{2^{32}}$。
4. **正弦查表**：预生成一个 1024 长度的 `uint16_t` 类型的正弦波映射表（Sine LUT），映射值范围应匹配 TIM1 的 ARR 寄存器范围（假设为 0 ~ ARR_MAX）。

# 核心架构要求 (严格遵守)
1. **纯 DMA 驱动**：严禁使用 CPU 中断逐个更新占空比。必须使用 DMA 将内存中的占空比数组搬运至 `TIM1->CCR1`。触发源为 TIM1 的更新事件 (Update Event)。
2. **Ping-Pong 双缓冲**：定义一个大小为 `2000` 的 `uint16_t` 数组（即半缓冲区为 `1000` 个点，对应 10ms 中断一次）。开启 DMA 的半传输 (HT) 和全传输 (TC) 中断。
3. **频率切换无缝衔接（影子变量机制）**：
    - 必须使用 `Target_FTW`（目标频率控制字）和 `Current_FTW`（当前计算频率控制字）。
    - FreeRTOS 任务或其他控制逻辑只允许修改 `Target_FTW`。
    - 仅在 DMA 的 HT 或 TC 中断的**起始处**，允许执行 `if(Current_FTW != Target_FTW) Current_FTW = Target_FTW;`。
4. **脉冲计数驻留机制**：
    - 严禁使用 `HAL_Delay` 或 `osDelay` 控制扫频时间。
    - 在 DMA 中断内维护一个 `uint32_t pulse_counter`。每次进中断（HT或TC），该计数器增加半缓冲区的长度（1000）。
    - 当 `pulse_counter >= Target_Pulse_Count` 时，切换到扫频表（Sweep Table）的下一个频点，更新 `Target_FTW` 和 `Target_Pulse_Count`，并将 `pulse_counter` 清零。
5. **H7 Cache 一致性**：在 DMA 中断内，CPU 计算并填充完半个缓冲区后，**必须**调用 `SCB_CleanDCache_by_Addr()` 清除该段内存的 D-Cache，否则 DMA 无法搬运最新数据。
